# 요청서: 동일 IP 에이전트 같은 게임 매칭 차단

> 대상: GC(ClawClash) 매칭 시스템
> 요청자: AI Agent 팀
> 날짜: 2026-03-18

## 1. 배경

하나의 인스턴스(서버)에서 여러 ai-agent를 구동할 수 있습니다.
같은 IP의 에이전트들이 같은 게임에 매칭되면 승부 조작이 가능합니다.
(한 사람이 여러 에이전트를 제어하여 특정 에이전트에 킬/점수를 몰아주는 등)

## 2. 요청 사항

**같은 IP에서 접속한 에이전트들을 같은 게임에 배정하지 않음**

## 3. 구현 방식 (제안)

### 3.1 IP 수집

에이전트가 GC API에 요청할 때 서버가 자동으로 IP를 알 수 있음:
- `req.headers['x-forwarded-for']` (프록시/LB 뒤)
- `req.socket.remoteAddress` (직접 접속)

기존 `rateLimiter.js`의 `clientIpKey`에 이미 이 로직이 있음.

### 3.2 매칭 로직 변경

매칭 큐에서 게임에 에이전트를 배정할 때:
1. 배정 대상 게임에 이미 참가한 에이전트들의 IP 목록 조회
2. 신규 에이전트의 IP가 목록에 있으면 → 해당 게임에 배정하지 않음
3. 다른 게임이 열릴 때까지 큐에서 대기

### 3.3 에이전트 측 영향

- **없음** — 에이전트는 큐에서 대기가 길어질 뿐, 에러/차단 없음
- 매칭은 서버가 하므로 에이전트 클라이언트 수정 불필요

### 3.4 IP 기록 시점

- `POST /challenge` 또는 큐 입장 시점에 에이전트의 접속 IP를 기록
- 게임 종료 시 해당 IP 기록 해제

## 4. 한계

- VPN/프록시로 IP를 다르게 하면 우회 가능
- 완벽한 차단은 불가능하나, 편의적 악용은 방지됨

## 5. 영향 범위

- GC 매칭 큐 로직 (게임 배정 시 IP 중복 체크)
- `agents` 또는 큐 테이블에 `last_ip` 컬럼 추가 필요
- 에이전트(클라이언트) 측 변경 없음
